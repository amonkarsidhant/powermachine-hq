# Learning Sprint Playbooks

Each sprint is 1 week long with a theme for the squads. Track completion via GitHub issues or a shared note in the Control UI workspace.

## Sprint 1 — "Autonomy Foundations" (Week 1)
**Focus**: Understand OpenClaw + Ollama orchestration and token management.
- **Playlists/articles**:
  - YouTube: "OpenClaw v2026 overview" (build story + Control UI). Create a summary doc.
  - Blog: "Designing multi-agent workflows" (15-min read from Opencode). Capture takeaways in Notion.
- **Expected outcome**: An agent skill template (prompt + tool stack) and an open issue for token budget automation.
- **Sprint owner**: AI Architect + Developer Experience engineers.

## Sprint 2 — "Reliable Platform" (Week 2)
**Focus**: Infrastructure, CI/CD, and monitoring.
- **Playlists/articles**:
  - SRE YouTube: "Production readiness for AI orchestration" (Google Cloud SRE session, reuse generalizable points).<br>
  - Article: "Chaos engineering with systemd + ngrok" (Cloudflare blog).
- **Outcome**: Documented workflow for ngrok/proxy monitoring, Github workflow drafted, base alerts defined.
- **Owners**: Platform Engineers + SRE.

## Sprint 3 — "Signal + Strategy" (Week 3)
**Focus**: Product discovery, quality assurance, and agent alignment.
- **Playlists/articles**:
  - YouTube: "Product management for AI-first teams" (re: Atlassian team).<br>
  - Article: "Agentic QA frameworks" (Medium / Rough Draft).<br>
  - Podcast: choose one from "AI in Production" by O'Reilly.
- **Outcome**: Product brief template, QA rubric, scoreboard for agent performance.
- **Owners**: Product, QA, Chaos.

## Execution tips
- Each sprint ends with a 30-min review posted in GitHub Discussions (or a note under `LEARNING-SPRINTS.md`).
- Use the Control UI for summary notes; pin the sprint doc for quick access.
- Auto-archive timestamps for future retrospectives.
